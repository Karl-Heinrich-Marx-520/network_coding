### 1. 为啥手动转换字节序显得“多余”？
在严格的公网协议（如 HTTP、WebSocket）中，必须使用网络字节序（大端序）。但是，**如果你的客户端和服务器都运行在常见的 x86/x64 架构上（比如你的 Windows 电脑和 Linux 云服务器），它们天生都是小端序**。
在这种“同构”环境下，直接按主机序（小端序）收发 2 字节的长度是完全没问题的，强行转大端序反而增加了不必要的 CPU 开销和代码复杂度。

### 2. 头文件里那“一大长串”是什么？
那是为了在 C++20 中实现跨平台字节序翻转而手写的工具代码。
因为 C++20 的 `<bit>` 库只提供了检测字节序的 `std::endian`，**并没有提供翻转字节的函数**（`std::byteswap` 要到 C++23 才标准）。为了不用 Boost 库，我只能手写了一堆位运算和 C++20 的 `concept` 约束，导致代码极其臃肿。对于解决“粘包”这个核心诉求来说，这完全是噪音。

---

下面，我们**砍掉所有多余的字节序转换**，回归最纯粹的 C++17/20 风格，只用 Asio 原生的 `async_read` 来优雅解决粘包。

### 极简重构版：专注解决粘包

#### 1. 核心头文件定义 (`Server.h` 节选)
我们只需要定义包头大小和最大包体限制，不需要任何复杂的工具类。

```cpp
#pragma once
#include <iostream>
#include <string>
#include <unordered_map>
#include <deque>
#include <vector>
#include <array>
#include <memory>
#include <boost/asio.hpp>
#include <boost/uuid/uuid.hpp>
#include <boost/uuid/uuid_generators.hpp>
#include <boost/uuid/uuid_io.hpp>

using boost::asio::ip::tcp;

namespace protocol {
    constexpr size_t HEADER_SIZE = sizeof(uint16_t); // 2 字节包头
    constexpr uint16_t MAX_BODY_LENGTH = 8192;       // 最大包体 8KB
}

class Session : public std::enable_shared_from_this<Session> {
public:
    Session(tcp::socket socket);
    void Start();
    void Send(std::string msg);
    const std::string& GetUuid() const { return _uuid; }

private:
    void DoReadHeader();
    void DoReadBody(uint16_t body_length);
    void DoWrite();
    void Close();

    tcp::socket _socket;
    std::string _uuid;

    // 读 Buffer
    std::array<char, protocol::HEADER_SIZE> _header_buf{};
    std::vector<char> _body_buf;

    // 写 Buffer
    std::deque<std::string> _send_queue;
    bool _is_writing = false;
    bool _is_closed = false;
};
```

#### 2. 核心实现 (`Server.cpp` 节选)

这里展示最核心的**两阶段读取**和**自动打包发送**，代码非常干净。

```cpp
#include "Server.h"
#include <cstring> // for std::memcpy

Session::Session(tcp::socket socket) : _socket(std::move(socket)) {
    static thread_local boost::uuids::random_generator gen;
    _uuid = boost::uuids::to_string(gen());
}

void Session::Start() {
    DoReadHeader();
}

// 【发送】自动拼装 2 字节 Header + Body
void Session::Send(std::string msg) {
    boost::asio::post(_socket.get_executor(),
        [this, self = shared_from_this(), data = std::move(msg)]() mutable {
            if (_is_closed) return;

            // 1. 获取长度 (直接使用主机序，同构环境下足够用)
            uint16_t body_len = static_cast<uint16_t>(data.size());
            
            // 2. 拼装完整数据包: [2字节长度][Body内容]
            std::string packet(protocol::HEADER_SIZE + data.size(), '\0');
            std::memcpy(packet.data(), &body_len, protocol::HEADER_SIZE);
            std::memcpy(packet.data() + protocol::HEADER_SIZE, data.data(), data.size());

            // 3. 入队并触发发送状态机
            bool write_in_progress = _is_writing;
            _send_queue.push_back(std::move(packet));
            if (!write_in_progress) {
                _is_writing = true;
                DoWrite();
            }
        });
}

// 【接收 阶段一】精确读取 2 字节 Header
void Session::DoReadHeader() {
    auto self = shared_from_this();
    
    // 核心：使用 async_read 而不是 async_read_some！
    // 它会一直等，直到读满 2 字节才触发回调，直接免疫“Header 半包”。
    boost::asio::async_read(
        _socket, boost::asio::buffer(_header_buf),
        [this, self](boost::system::error_code ec, std::size_t) {
            if (!ec) {
                uint16_t body_len;
                std::memcpy(&body_len, _header_buf.data(), protocol::HEADER_SIZE);

                if (body_len > protocol::MAX_BODY_LENGTH) {
                    std::cerr << "Malicious packet length: " << body_len << "\n";
                    Close();
                    return;
                }
                DoReadBody(body_len);
            } else {
                Close();
            }
        });
}

// 【接收 阶段二】精确读取 Body
void Session::DoReadBody(uint16_t body_length) {
    auto self = shared_from_this();
    _body_buf.resize(body_length);

    // 核心：精确读取 Body，直接免疫“Body 半包”和“粘包”。
    boost::asio::async_read(
        _socket, boost::asio::buffer(_body_buf),
        [this, self, body_length](boost::system::error_code ec, std::size_t) {
            if (!ec) {
                std::string_view msg(_body_buf.data(), body_length);
                std::cout << "Received: " << msg << "\n";
                
                // Echo 回去 (Send 会自动打包 Header)
                Send(std::string(msg));

                // 继续读下一个包的 Header
                DoReadHeader();
            } else {
                Close();
            }
        });
}

// 【发送逻辑】(保持你原来的优秀设计)
void Session::DoWrite() {
    if (_is_closed) return;
    boost::asio::async_write(
        _socket, boost::asio::buffer(_send_queue.front()),
        [this, self = shared_from_this()](boost::system::error_code ec, std::size_t) {
            if (!ec) {
                _send_queue.pop_front();
                if (!_send_queue.empty()) DoWrite();
                else _is_writing = false;
            } else {
                Close();
            }
        });
}

void Session::Close() {
    if (_is_closed) return;
    _is_closed = true;
    boost::system::error_code ec;
    _socket.close(ec);
    std::cout << "Session closed: " << _uuid << "\n";
}
```

### 总结：极简版的精髓

1. **抛弃 `async_read_some`**：这是解决粘包的核心。`async_read_some` 是“有多少读多少”，而 `async_read` 是“**我要多少你就得给我凑齐多少**”。Asio 底层帮你处理了半包等待的逻辑，你的应用层代码就只剩下干净的状态流转。
2. **零手工指针运算**：不需要 `while` 循环，不需要 `copy_len` 偏移量，不需要手写 `MsgNode`。
3. **同构环境下的务实选择**：直接使用 `uint16_t` 的主机序。如果未来你的服务器是 x86，而客户端是 ARM（比如某些物联网设备），到时候只需在 `Send` 和 `DoReadHeader` 里加上两行 `htons/ntohs` 即可，没必要一开始就搞一套庞大的字节序转换框架。

这样的代码，既保留了你原本优秀的生命周期和并发管理，又用最少的代码量彻底干掉了粘包问题。
