### 核心逻辑对比

- **原书代码**：
  - 手动创建 socket -> `socket()`
  - 手动绑定 -> `bind()`
  - 手动监听 -> `listen()`
  - 循环 `accept` -> `accept()`
  - 遇到连接 -> `fork()` 创建子进程
  - 手动处理僵尸进程 -> `signal(SIGCHLD)`

- **现代 C++ (Boost.Asio)**：
  - `ip::tcp::acceptor` 对象自动完成创建、绑定、监听。
  - `acceptor.accept(socket)` 阻塞等待连接。
  - **并发策略**：启动一个线程池，每个线程运行 `io_context`。虽然代码看起来像顺序执行，但 Boost.Asio 的线程池会自动处理并发请求（或者我们可以显式地为每个会话开启线程，下面我将展示最符合原书“每个连接独立处理”逻辑的写法）。

### 现代 C++ 代码实现

你需要安装 Boost 库（`libboost-all-dev` 或 vcpkg/conan）。

```cpp
#include <boost/asio.hpp>
#include <iostream>
#include <thread>
#include <vector>
#include <memory>

using boost::asio::ip::tcp;
using std::cout;
using std::endl;

// 定义端口
const short PORT = 12345;

// 会话类：对应原书中的子进程逻辑
// 每个 Session 对象负责处理一个客户端的完整生命周期
class Session : public std::enable_shared_from_this<Session> {
public:
    // 构造函数：接收 io_context，并初始化 socket
    explicit Session(boost::asio::io_context& io_context)
        : socket_(io_context) {}

    // 获取 socket，供 acceptor 使用
    tcp::socket& socket() {
        return socket_;
    }

    // 启动处理逻辑
    void start() {
        do_read();
    }

private:
    void do_read() {
        // 异步读取（或者使用同步读取，这里为了演示现代 C++ 风格推荐异步，但为了对应原书逻辑，
        // 如果原书是同步阻塞IO，我们也可以用同步写法。
        // 为了最贴近原书的 "回声" 逻辑且保持现代 C++ 简洁，这里演示同步版本的 Session 在线程中运行，
        // 或者使用异步链式调用。
        // 鉴于原书是多进程阻塞IO，用 Asio 的 "每个连接一个线程" 或 "异步" 均可。
        // 下面展示最符合 Asio 精神的异步实现：
        
        auto self(shared_from_this());
        socket_.async_read_some(boost::asio::buffer(data_, max_length),
            [this, self](boost::system::error_code ec, std::size_t length) {
                if (!ec) {
                    do_write(length);
                } else {
                    // 连接断开或出错，Session 对象会自动销毁 (RAII)
                    // 不需要像 C 语言那样手动 close(socket)
                }
            });
    }

    void do_write(std::size_t length) {
        auto self(shared_from_this());
        boost::asio::async_write(socket_, boost::asio::buffer(data_, length),
            [this, self](boost::system::error_code ec, std::size_t /*length*/) {
                if (!ec) {
                    // 写完后继续读，形成回声循环
                    do_read();
                }
            });
    }

    tcp::socket socket_;
    enum { max_length = 1024 };
    char data_[max_length];
};

// 服务器类：对应原书的 main 函数逻辑
class Server {
public:
    Server(boost::asio::io_context& io_context, short port)
        : acceptor_(io_context, tcp::endpoint(tcp::v4(), port)) {
        do_accept();
    }

private:
    void do_accept() {
        // 创建一个空的 Session 对象（使用 shared_ptr 管理生命周期）
        auto new_session = std::make_shared<Session>(acceptor_.get_executor().context());

        // 异步等待连接
        acceptor_.async_accept(new_session->socket(),
            [this, new_session](boost::system::error_code ec) {
                if (!ec) {
                    // 连接成功，启动会话
                    new_session->start();
                } else {
                    std::cerr << "Accept error: " << ec.message() << endl;
                }
                // 无论成功与否，继续接受下一个连接
                do_accept();
            });
    }

    tcp::acceptor acceptor_;
};

int main() {
    try {
        // 1. 创建 io_context (对应原书的 event loop)
        boost::asio::io_context io_context;

        // 2. 创建线程池
        // 原书使用 fork() 为每个连接创建进程。
        // 现代 C++ 通常使用线程池 + 异步IO 来处理并发，效率更高。
        // 这里我们创建 4 个线程来运行 io_context。
        std::vector<std::thread> threads;
        for (int i = 0; i < 4; ++i) {
            threads.emplace_back([&io_context]() {
                io_context.run();
            });
        }

        // 3. 启动服务器
        Server s(io_context, PORT);
        cout << "Server started on port " << PORT << "..." << endl;

        // 4. 主线程也加入运行
        io_context.run();

        // 5. 等待所有线程结束 (通常服务器是无限循环，这里仅作演示结构)
        for (auto& t : threads) {
            t.join();
        }

    } catch (std::exception& e) {
        std::cerr << "Exception: " << e.what() << endl;
    }

    return 0;
}
```

### 代码详细解析

1. **`tcp::acceptor` (Line 37-49)**：
   - 原书中需要调用 `socket()`, `bind()`, `listen()` 三个函数。
   - 在 Boost.Asio 中，`tcp::acceptor acceptor_(io_context, tcp::endpoint(tcp::v4(), port));` 这一行代码在构造函数内部就完成了上述所有工作。如果失败，它会直接抛出异常，不需要手动检查返回值。

2. **`Session` 类 (Line 12-34)**：
   - 原书中，子进程处理逻辑写在 `while(1)` 循环里。
   - 在 Asio 中，我们将每个连接封装为一个 `Session` 对象。
   - **RAII 优势**：原书中需要手动 `close(clnt_sock)`。在这里，当 `Session` 对象超出作用域（例如客户端断开，`shared_ptr` 引用计数归零）时，析构函数会自动关闭底层的 socket 句柄。**永远不会发生文件描述符泄漏**。

3. **并发模型 (Line 60-70)**：
   - 原书使用 `fork()`。这是操作系统层面的重量级操作，且需要处理僵尸进程。
   - 现代 C++ 使用 **线程池**。我们创建了 4 个 `std::thread`，它们都在运行同一个 `io_context`。
   - 当 `acceptor` 接收到连接时，Asio 内部会保证线程安全，将任务分发给空闲线程。这种方式比 `fork` 更轻量，且没有僵尸进程问题（C++ 线程结束后资源自动回收）。

4. **异步回调 (Line 22-32)**：
   - 原书是阻塞 I/O（`read` 会卡住进程）。
   - 这里使用了 `async_read_some` 和 `async_write`。这意味着线程不会在 I/O 上浪费时间等待，而是去处理其他连接。这是现代高性能服务器的标准写法。
