在IPv4网络中，IP地址由网络ID和主机ID组成，用于唯一标识网络中的设备。公司局域网中的计算机通常处于同一子网内，其IP地址的网络ID相同，主机ID不同。数据在局域网内传输时，主要依赖数据链路层的MAC地址进行寻址，而路由器在此场景下主要承担网关角色，负责连接不同网络或子网。

当源计算机向局域网内另一台目标计算机发送数据时：

1. IP地址解析：源主机根据目标IP地址判断其是否在同一网络（即网络ID相同）。若在同一网络，则直接通过ARP协议获取目标主机的MAC地址。

2. 数据封装：源主机将数据封装成以太网帧，帧头中包含源MAC地址和目标MAC地址，IP数据包中包含源IP地址和目标IP地址。

3. 交换机转发：局域网内的交换机根据目标MAC地址表将帧转发至目标主机所在端口。

4. 路由器作用：若目标主机位于不同子网，则数据包需经过路由器。路由器根据IP数据包中的目的网络地址查找路由表，决定下一跳路径，并将数据包转发至目标网络。在局域网内部，若所有主机在同一子网，则无需路由器参与转发，仅需交换机完成数据链路层转发。

综上，在局域网内部，数据主要依靠网络ID判断是否在同一网段，通过主机ID定位具体设备，而路由器则负责跨网段的数据转发，其核心依据是目的网络地址。

答案： 

在局域网内，IP地址的网络ID用于判断目标主机是否在同一子网，主机ID用于定位具体设备。若目标在同一子网，数据通过ARP获取MAC地址后，由交换机依据MAC地址表转发至目标主机；若目标在不同子网，则数据包经路由器，路由器根据目的网络地址查路由表并转发至下一跳，最终送达目标主机。
