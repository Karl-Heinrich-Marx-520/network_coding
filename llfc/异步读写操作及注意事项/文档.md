# 现代 C++ Boost.Asio 异步网络编程指南

## 简介
本文介绍 Boost.Asio 的异步读写操作及注意事项。为保证知识便于吸收，本节仅介绍 API 使用的代码片段，下一节再编写完整的客户端和服务器程序。

首先，我们定义一个 `Session` 类。在现代 C++ 中，`Session` 代表服务器与特定客户端之间的一次完整会话（连接）的生命周期。

```cpp
#include <memory>
#include <boost/asio.hpp>

namespace asio = boost::asio;
using tcp = asio::ip::tcp;

class Session {
public:
    // 【现代 C++ 重构】：使用 std::unique_ptr 明确表达“独占所有权”。
    // 原写法使用 std::shared_ptr，但在单一 Session 管理单一 Socket 的场景下，
    // unique_ptr 语义更准确，且没有引用计数的性能开销。
    explicit Session(std::unique_ptr<tcp::socket> socket) 
        : _socket(std::move(socket)) {}

    void Connect(const tcp::endpoint& ep);

private:
    std::unique_ptr<tcp::socket> _socket;
};

void Session::Connect(const tcp::endpoint& ep) {
    // 【现代 C++ 重构】：使用 std::error_code 替代抛出异常，避免程序意外崩溃
    boost::system::error_code ec;
    _socket->connect(ep, ec);
    if (ec) {
        // 记录日志或处理错误，而不是直接 perror/exit
    }
}
```
*这里只是简单演示连接，下面核心介绍异步读写 API 的使用。*

## 异步写操作

在写操作前，我们需要封装一个 `MsgNode` 结构，用来管理要发送和接收的数据。它包含数据载体、总长度以及已处理的长度。

**【重构点】**：原代码使用 `new char[]` 和手动 `delete[]`，这是 C 语言时代的做法，极易造成内存泄漏。现代 C++ 中，我们使用 `std::vector<char>` 作为底层缓冲区，它自带 RAII（资源获取即初始化）特性，析构时自动释放内存。

```cpp
#include <vector>
#include <string>
#include <string_view>

// 最大报文接收大小
constexpr std::size_t RECV_SIZE = 1024;

class MsgNode {
public:
    // 发送时构造函数：从 string_view 拷贝数据 (C++17 特性，比 const string& 更通用且无拷贝开销)
    explicit MsgNode(std::string_view msg) 
        : _data(msg.begin(), msg.end()), _cur_len(0) {}

    // 接收时构造函数：预分配指定大小的空间
    explicit MsgNode(std::size_t total_len) 
        : _data(total_len), _cur_len(0) {}

    // 获取底层数据指针，供 Asio buffer 使用
    char* data() { return _data.data(); }
    std::size_t total_len() const { return _data.size(); }
    
    std::size_t _cur_len; // 当前已处理的长度

private:
    std::vector<char> _data; // 【现代 C++ 替代】裸露的 char* _msg
};
```

接下来为 `Session` 添加基础的异步写操作。

```cpp
class Session {
public:
    void WriteCallBackErr(const boost::system::error_code& ec, 
                          std::size_t bytes_transferred,
                          std::shared_ptr<MsgNode> msg_node);
    void WriteToSocketErr(const std::string& buf);
private:
    std::unique_ptr<tcp::socket> _socket;
    std::shared_ptr<MsgNode> _send_node; 
};
```

### 为什么使用 Lambda 替代 std::bind？
Asio 的 `async_write_some` 要求的回调签名是 `void(error_code, size_t)`。原代码使用 `std::bind` 来绑定额外的参数。在现代 C++ 中，**强烈建议使用 Lambda 表达式**。`std::bind` 会产生复杂的模板推导开销，且不支持移动语义；而 Lambda 可读性更强，类型推导更精准。

```cpp
void Session::WriteToSocketErr(const std::string& buf) {
    // 使用 std::make_shared 现代工厂函数，异常安全且只需一次内存分配
    auto node = std::make_shared<MsgNode>(buf);
    _send_node = node; 

    // 发起异步发送
    _socket->async_write_some(
        asio::buffer(node->data(), node->total_len()),
        // 【现代 C++ 核心】：Lambda 表达式
        // 按值捕获 this 和 node。
        // 捕获 node (shared_ptr) 会增加引用计数，确保在异步回调触发前，
        // MsgNode 的内存绝对不会被释放！这是解决 Asio 异步生命周期问题的标准做法。
        [this, node](const boost::system::error_code& ec, std::size_t bytes_transferred) {
            this->WriteCallBackErr(ec, bytes_transferred, node);
        }
    );
}
```

### 处理“半包”问题 (async_write_some)
`async_write_some` 不保证一次性发完所有数据。如果 TCP 发送缓冲区满了，它只会发送一部分并触发回调。我们需要在回调中计算剩余长度，继续发送。

```cpp
void Session::WriteCallBackErr(const boost::system::error_code& ec, 
                               std::size_t bytes_transferred,
                               std::shared_ptr<MsgNode> msg_node) {
    // 【现代 C++ 重构】：直接判断 ec 的布尔值，无需 .value() != 0
    if (ec) {
        return;
    }

    // 更新已发送长度
    msg_node->_cur_len += bytes_transferred;

    // 判断是否还有剩余数据未发送
    if (msg_node->_cur_len < msg_node->total_len()) {
        // 计算剩余数据的起始指针和长度
        char* remaining_data = msg_node->data() + msg_node->_cur_len;
        std::size_t remaining_len = msg_node->total_len() - msg_node->_cur_len;

        // 继续异步发送剩余部分
        _socket->async_write_some(
            asio::buffer(remaining_data, remaining_len),
            // 递归调用自身，继续按值捕获 msg_node 维持生命周期
            [this, msg_node](const boost::system::error_code& next_ec, std::size_t next_bytes) {
                this->WriteCallBackErr(next_ec, next_bytes, msg_node);
            }
        );
    }
}
```

### 解决消息交织问题（引入发送队列）
上面的代码在实际应用中是**危险**的。如果用户连续调用两次 `WriteToSocketErr`，而第一次的数据还没发完（回调还没触发），第二次调用会覆盖成员变量 `_send_node`，导致第一次回调时读取到错误的数据，最终对端收到的消息会交织错乱。

**解决方案**：引入应用层发送队列，保证消息按顺序发送。

```cpp
#include <queue>

class Session {
public:
    void WriteCallBack(const boost::system::error_code& ec, std::size_t bytes_transferred);
    void WriteToSocket(std::string_view buf);
private:
    std::unique_ptr<tcp::socket> _socket;
    
    // 现代 C++ 队列：存储智能指针，自动管理内存
    std::queue<std::shared_ptr<MsgNode>> _send_queue; 
    bool _send_pending = false; // 标记是否有正在发送的任务
};

void Session::WriteToSocket(std::string_view buf) {
    // 【重构】使用 std::make_shared 替代 new，防止异常导致内存泄漏
    _send_queue.push(std::make_shared<MsgNode>(buf));

    // pending 状态说明上一次有未发送完的数据，直接返回等待回调处理
    if (_send_pending) {
        return;
    }

    // 【致命 Bug 修复】：原代码使用 asio::buffer(buf)，如果 buf 是临时变量，
    // 异步操作还没发完，buf 就销毁了，导致读取野指针崩溃！
    // 正确做法：必须使用队列中 MsgNode 拥有的内存作为 buffer。
    auto& front_node = _send_queue.front();
    
    _socket->async_write_some(
        asio::buffer(front_node->data(), front_node->total_len()),
        [this](const boost::system::error_code& ec, std::size_t bytes_transferred) {
            this->WriteCallBack(ec, bytes_transferred);
        }
    );
    _send_pending = true;
}

void Session::WriteCallBack(const boost::system::error_code& ec, std::size_t bytes_transferred) {
    if (ec) {
        _send_pending = false;
        return;
    }

    auto& send_data = _send_queue.front();
    send_data->_cur_len += bytes_transferred;

    // 1. 数据未发送完，则继续发送
    if (send_data->_cur_len < send_data->total_len()) {
        _socket->async_write_some(
            asio::buffer(send_data->data() + send_data->_cur_len, 
                         send_data->total_len() - send_data->_cur_len),
            [this](const boost::system::error_code& next_ec, std::size_t next_bytes) {
                this->WriteCallBack(next_ec, next_bytes);
            }
        );
        return;
    }

    // 2. 当前消息发送完毕，出队（shared_ptr 自动释放 MsgNode 内存）
    _send_queue.pop();

    // 3. 检查队列是否还有后续消息
    if (_send_queue.empty()) {
        _send_pending = false; // 队列空，标记为空闲
    } else {
        // 队列不为空，继续发送下一个队首消息
        auto& next_node = _send_queue.front();
        _socket->async_write_some(
            asio::buffer(next_node->data() + next_node->_cur_len, 
                         next_node->total_len() - next_node->_cur_len),
            [this](const boost::system::error_code& next_ec, std::size_t next_bytes) {
                this->WriteCallBack(next_ec, next_bytes);
            }
        );
    }
}
```

### 终极简化：使用 `asio::async_write` 
*注：原文档中提到的 `async_send` 在标准 Asio 中并不存在，正确的保证完整发送的自由函数是 `asio::async_write`。*

手动在回调中计算偏移量非常繁琐。Asio 提供了自由函数 `asio::async_write`，它内部会自动循环调用 `async_write_some`，**直到所有数据发送完毕或发生错误才会触发回调**。这能大幅简化我们的代码。

```cpp
#include <boost/asio/write.hpp> // 必须包含此头文件

class Session {
public:
    void WriteAllCallBack(const boost::system::error_code& ec, std::size_t bytes_transferred);
    void WriteAllToSocket(std::string_view buf);
private:
    std::queue<std::shared_ptr<MsgNode>> _send_queue;
    std::unique_ptr<tcp::socket> _socket;
    bool _send_pending = false;
};

void Session::WriteAllToSocket(std::string_view buf) {
    _send_queue.push(std::make_shared<MsgNode>(buf));
    if (_send_pending) return;

    auto& front_node = _send_queue.front();
    
    // 【现代 Asio 推荐】：使用 asio::async_write 保证完整发送
    asio::async_write(
        *_socket, 
        asio::buffer(front_node->data(), front_node->total_len()),
        [this](const boost::system::error_code& ec, std::size_t bytes_transferred) {
            this->WriteAllCallBack(ec, bytes_transferred);
        }
    );
    _send_pending = true;
}

void Session::WriteAllCallBack(const boost::system::error_code& ec, std::size_t bytes_transferred) {
    if (ec) {
        _send_pending = false;
        return;
    }
    
    // 触发回调时，保证整个 MsgNode 已经发送完毕，直接出队即可
    _send_queue.pop();

    if (_send_queue.empty()) {
        _send_pending = false;
    } else {
        auto& next_node = _send_queue.front();
        asio::async_write(
            *_socket,
            asio::buffer(next_node->data(), next_node->total_len()),
            [this](const boost::system::error_code& next_ec, std::size_t next_bytes) {
                this->WriteAllCallBack(next_ec, next_bytes);
            }
        );
    }
}
```
*注意：`asio::async_write` 和 `async_write_some` 绝对不能混用，否则会导致底层状态机混乱。*

## 异步读操作

异步读操作与写操作类似，同样存在 `async_read_some`（可能读不完）和 `asio::async_read`（保证读完指定长度，*原文档误写为 async_receive*）的区别。

```cpp
class Session {
public:
    void ReadFromSocket();
    void ReadCallBack(const boost::system::error_code& ec, std::size_t bytes_transferred);
    
    void ReadAllFromSocket();
    void ReadAllCallBack(const boost::system::error_code& ec, std::size_t bytes_transferred);

private:
    std::unique_ptr<tcp::socket> _socket;
    std::shared_ptr<MsgNode> _recv_node;
    bool _recv_pending = false;
};
```

### 基础读取：处理“半包” (async_read_some)
如果不考虑应用层协议粘包，仅按固定字节接收，我们需要手动处理未读完的情况。

```cpp
void Session::ReadFromSocket() {
    if (_recv_pending) return;

    _recv_node = std::make_shared<MsgNode>(RECV_SIZE);
    
    _socket->async_read_some(
        asio::buffer(_recv_node->data(), _recv_node->total_len()),
        [this](const boost::system::error_code& ec, std::size_t bytes_transferred) {
            this->ReadCallBack(ec, bytes_transferred);
        }
    );
    _recv_pending = true;
}

void Session::ReadCallBack(const boost::system::error_code& ec, std::size_t bytes_transferred) {
    if (ec) {
        _recv_pending = false;
        _recv_node.reset(); // 【现代 C++ 替代】 _recv_node = nullptr，显式释放
        return;
    }

    _recv_node->_cur_len += bytes_transferred;

    // 没读完，继续读剩余部分
    if (_recv_node->_cur_len < _recv_node->total_len()) {
        _socket->async_read_some(
            asio::buffer(_recv_node->data() + _recv_node->_cur_len, 
                         _recv_node->total_len() - _recv_node->_cur_len),
            [this](const boost::system::error_code& next_ec, std::size_t next_bytes) {
                this->ReadCallBack(next_ec, next_bytes);
            }
        );
        return;
    }

    // 读完了，将数据投递到业务逻辑队列（此处略去）
    _recv_pending = false;
    _recv_node.reset(); 
}
```

### 终极简化：使用 `asio::async_read`
与写操作一样，Asio 提供了 `asio::async_read`，它会一直读，直到填满 Buffer 或发生错误才触发回调。

```cpp
#include <boost/asio/read.hpp> // 必须包含此头文件

void Session::ReadAllFromSocket() {
    if (_recv_pending) return;

    _recv_node = std::make_shared<MsgNode>(RECV_SIZE);

    // 【现代 Asio 推荐】：使用 asio::async_read 保证读取完整长度
    asio::async_read(
        *_socket,
        asio::buffer(_recv_node->data(), _recv_node->total_len()),
        [this](const boost::system::error_code& ec, std::size_t bytes_transferred) {
            this->ReadAllCallBack(ec, bytes_transferred);
        }
    );
    _recv_pending = true;
}

void Session::ReadAllCallBack(const boost::system::error_code& ec, std::size_t bytes_transferred) {
    if (ec) {
        _recv_pending = false;
        _recv_node.reset();
        return;
    }

    // 触发回调时，保证 RECV_SIZE 个字节已经全部读完
    // 将数据投递到业务逻辑队列（此处略去）
    
    _recv_pending = false;
    _recv_node.reset();
}
```

**总结**：在现代 C++ 与 Boost.Asio 的开发中，**RAII 容器（`std::vector`）+ 智能指针（`std::shared_ptr`）+ Lambda 按值捕获 + `asio::async_read/write` 自由函数**，是构建高并发、无内存泄漏网络程序的黄金组合。同样，`async_read_some` 和 `asio::async_read` 不可混用。
